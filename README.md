# SRE道場 — 落として直して強くなる

ローカルPCの中に**本物のサーバー**を建てて、**本物の負荷と攻撃**を撃ち込み、
落として、原因を掘って、直す。それを繰り返して **SRE（サービスを落とさない信頼性エンジニアリング）の判断力** を体で覚える演習場。

全部Dockerでローカル完結・無料。何度壊してもいい。自分のPCの中の自分のサーバーが相手なので安全で合法。

## 特徴
- **ブラウザのライブダッシュボード** — 今の構成図・各コンテナのCPU/メモリ・進行中の攻撃を1.5秒ごとに自動更新。構成を変えれば即反映。
- **ボタンで攻撃** — 4種類の攻撃（じわ増し／スパイク／重い処理／DB遅延）をワンクリックで発射。パターンごとに壊れる場所が変わる。
- **画面内ログビューア** — 各コンテナのログをブラウザで表示、エラー行を赤くハイライト。ターミナル不要。
- **難易度トグル** — 「易しい」でメトリクスとログの読み方を解説。ただし答え（診断）は出さない＝自分で推測する設計。
- **依存ゼロ** — ダッシュボードはPython標準ライブラリのみ。要るのはDockerだけ。

## ライブダッシュボード（推奨の入口）
```powershell
./dojo.ps1 up          # 第1章を起動
./dojo.ps1 dashboard   # http://localhost:8090 が開く
```
あとはブラウザのボタンで攻撃を撃ち、バーの動きとログを見て「どこが詰まったか」を推測する。

## 1戦のループ
1. **建てる** — 素朴な構成を起動する
2. **攻める** — k6で負荷を、攻撃スクリプトで攻撃を撃ち込む
3. **観る** — `docker stats` とログで「どこが詰まったか」を見る
4. **診る** — CPUか、メモリか、DBの接続数か、ワーカー枯渇か。自分で当てる
5. **直す** — 設定を変える／台数を増やす／構成を変える
6. **もう一段強い攻撃へ** — 直したら次のボトルネックが必ず出てくる（これが歴史そのもの）

## 章立て（物理 → VM → クラウド）
- **第1章 物理時代** (`chapter1-physical/`) — 貧弱な1台(nginx/php+mysql)。全部1台に乗せて、負荷で落とす。ボトルネックを1つずつ潰し、スケールアップの限界を体感する。
- **第2章 VM時代** (`chapter2-vm/`) — 複数台に増やし、**LBの設定を自分で書く**。1台をkillして可用性を体感。DBボトルネック・セッション問題。
- **第3章 クラウド時代** (`chapter3-cloud/`) — **オートスケーラを自作**（CPUを見て台数を増やす数十行）。書き終えた瞬間「Auto Scalingの正体」が分かる。ここまで来たら本物のAWSへ卒業。

## 使い方（PowerShellから）
```powershell
# 第1章を起動
./dojo.ps1 up
# ブラウザで見る（攻撃中に重くなる/落ちるのが見える）
./dojo.ps1 open
# リソースをライブ監視（別ウィンドウで開きっぱなしにする）
./dojo.ps1 stats
# 負荷攻撃を撃ち込む
./dojo.ps1 attack
# ログを見る（"Too many connections" 等の悲鳴がここに出る）
./dojo.ps1 logs
# 片付け
./dojo.ps1 down
```
別の章を触るときは `-Chapter chapter2-vm` のように渡す。

## 前提
- Docker Desktop が起動していること（`docker version` が通る状態）
- ダッシュボードに Python 3（標準ライブラリのみ、追加インストール不要）
- k6・mysql・php は全部コンテナで動くので、個別インストールは不要
